    Финальное задание по дисциплине "Информационные системы и базы данных"

    Задание:

    Создать лабораторную работу по LLM-инструменту MongoDB с использованием большой языковой модели (например, DeepSeek).

    Промт для генерации лабораторной работы

    Ты — опытный методист и разработчик баз данных. Разработай детальный план лабораторной работы на тему «Работа с документоориентированной СУБД MongoDB» для студентов 3-го курса колледжа, обучающихся по специальности «Программная инженерия». Продолжительность работы — 4 академических часа.

    План лабораторной работы должен содержать следующие обязательные разделы:

    Цель работы: сформулировать конкретные практические навыки, которые студенты приобретут в ходе выполнения (CRUD-операции, построение агрегационных запросов, создание и анализ индексов).

    Теоретическое введение: кратко (в объёме 3–4 абзацев) описать ключевые отличия MongoDB от реляционных СУБД, особенности типа данных BSON и понятие коллекции.

    Инструментарий: представить два варианта работы со средой, каждый с пошаговым описанием: - Способ А (локальный): установка MongoDB Community Edition и MongoDB Compass. - Способ Б (облачный): регистрация в MongoDB Atlas (бесплатный тариф M0) как альтернатива локальной установке.

    Указать, что для выполнения запросов можно использовать как встроенную вкладку mongosh в MongoDB Compass, так и отдельный терминал.

    Предварительные действия: подробно описать шаги, которые студент обязан выполнить до начала написания запросов: - Запустить сервер MongoDB (или убедиться, что кластер Atlas активен). - Подключиться к серверу с использованием строки подключения. - Создать базу данных с именем library_db и внутри неё коллекцию books.

    Важно: если студент работает с локальным сервером, он должен предварительно создать каталог для хранения данных (например, data/db) или использовать команду mongod --dbpath. При работе с Atlas необходимо получить строку подключения из личного кабинета.

    Исходные данные: подготовить JSON-структуру документа для коллекции books, содержащую поля: title (строка), author (строка), year (число), genres (массив строк), price (число с плавающей точкой), rating (число от 1 до 5), stock (целое число, обозначающее количество экземпляров на складе). Сгенерировать 20 тестовых документов, имитирующих реальные книги, и предоставить готовый блок для вставки в коллекцию с использованием команды db.books.insertMany([...]).

    Ход работы (практические задания): разработать 5–7 заданий, охватывающих следующие темы: - Запросы find с фильтрацией с использованием операторов сравнения ($gt, $lt, $in), проекция полей, сортировка, пагинация через limit и skip. - Операции обновления: updateOne и updateMany с применением $set, $inc (для изменения цены), $push (добавление жанра в массив), $pull (удаление жанра из массива). Операции удаления: deleteOne по конкретному идентификатору и deleteMany по условию (например, удаление книг с нулевым остатком). - Построение конвейера агрегации, включающего этапы $match (фильтрация по году), $group (группировка по жанрам с подсчётом количества книг и вычислением средней цены), $sort (сортировка по количеству) и $project (переименование полей для улучшения читаемости вывода). - Создание составного индекса по полям author и year для ускорения поиска. Выполнить поиск с выводом плана выполнения через explain("executionStats") до создания индекса и после, чтобы студент наглядно увидел разницу в количестве просмотренных документов (totalDocsExamined).

    Методические рекомендации (важные пояснения): - В каждом задании должна быть чёткая постановка задачи, последовательность действий, требования к результату и ожидаемый вывод. - При использовании проекции во втором аргументе find нельзя одновременно указывать включение (1) и исключение (0), за исключением поля _id. Для вывода всех полей, кроме одного, следует использовать запись { field: 0 }. - Пояснить разницу между updateOne и updateMany: первый изменяет только первый найденный документ, даже если условию соответствует несколько. Для массового обновления необходимо явно применять updateMany. - Операторы $push и $pull применимы только к полям, имеющим тип массива; в противном случае возникнет ошибка. - В агрегационном конвейере этапы выполняются последовательно, и порядок следования $match и $group критически важен для производительности — фильтрацию данных следует производить как можно раньше.

    Контрольные вопросы для защиты (6 вопросов): - Четыре вопроса в формате теста с четырьмя вариантами ответов, с указанием правильного варианта в скобках после вопроса. - Два открытых вопроса, проверяющих понимание логики выполнения работы (например, сравнение вложенных документов и ссылок, либо принцип работы индексов в MongoDB).

    Самостоятельная работа (2 дополнительных задания): 1. Расширить схему коллекции books, добавив поле reviews — массив вложенных документов с полями user и comment. Написать запрос для поиска книг, содержащих отзыв конкретного пользователя, с использованием оператора $elemMatch. 2. Составить агрегационный запрос для подсчёта общего количества книг на складе (stock) в разрезе каждого автора и ранжирования авторов по этому показателю с использованием $sum.

    2. Теоретическое введение

    MongoDB — это документоориентированная система управления базами данных (NoSQL). В отличие от реляционных SQL БД, где информация хранится в таблицах со строго определённой структурой столбцов, MongoDB хранит данные в виде документов. Документы объединяются в коллекции.

    Документ MongoDB имеет структуру, похожую на JSON, но фактически хранится в формате BSON (Binary JSON). BSON поддерживает стандартные типы JSON, а также дополнительные типы данных: ObjectId, Date, Decimal128, Binary Data и другие.

    Коллекция MongoDB является аналогом таблицы в SQL. Однако в отличие от таблицы документы одной коллекции могут иметь разные поля. Например, один документ книги может содержать отзывы пользователей, а другой — нет.

    Главное преимущество документоориентированного подхода — удобное хранение сложных объектов с вложенными структурами и массивами. Например, информация о книге может содержать жанры, отзывы, характеристики и другие связанные данные внутри одного документа.

    3.1 Установка MongoDB

    Windows

    Linux

    После установки проверить:

    mongod --version

    macOS

    brew --version

    Установка:

    MongoDB требует каталог для хранения данных.

    Windows:

    Windows:

    mongod --dbpath C:\data\db

    Linux/macOS:

    mongod --dbpath /data/db

    Задание 10. Построение Aggregation Pipeline

    Постановка задачи

    Получить статистику:

    Этапы выполнения

    db.books.aggregate(

[

// Фильтрация документов перед обработкой
{
$match:
{
year:
{
$gt:1900
}
}
},

// Разворачиваем массив жанров
{
$unwind:"$genres"
},

// Группировка данных
{
$group:
{
_id:"$genres",

count:
{
$sum:1
},

avgPrice:
{
$avg:"$price"
}

}
},

// Сортировка групп
{
$sort:
{
count:-1
}
},

// Красивый вывод результата
{
$project:
{
Жанр:"$_id",
Количество:"$count",
СредняяЦена:"$avgPrice",
_id:0
}
}

]

)

    Требования

    Использовать:

    • $match;
    • $group;
    • $avg;
    • $sort;

    Методические рекомендации

    1. Правила проекции

    В MongoDB нельзя одновременно использовать включение и исключение:

    Нельзя:

    {
title:1,
price:0
}

    Исключение:

    _id:0

    разрешено.

    Если нужно вывести все поля кроме одного:

    Правильно:

    {
password:0
}

    2. Разница updateOne и updateMany

    updateOne():

    • изменяет только первый найденный документ;
    • даже если условию соответствует 10 документов.

    Пример:

    db.books.updateOne(
{
author:"Автор"
},
{
$set:
{
rating:5
}
}
)

    Изменится только одна книга.

    Для изменения всех используется:

    updateMany()

    3. Работа с массивами

    Операторы:

    $push
$pull

    работают только с массивами.

    Правильно:

    genres:[
"роман",
"фэнтези"
]

    Ошибка:

    genres:"роман"

    При использовании $push к строке MongoDB выдаст ошибку типа данных.

    4. Порядок этапов агрегации

    Aggregation Pipeline выполняется строго сверху вниз.

    Неправильно:

    $group
↓
$match

    Правильно:

    $match
↓
$group

    Фильтрацию необходимо выполнять как можно раньше.

    Это уменьшает количество документов для обработки и повышает производительность.
