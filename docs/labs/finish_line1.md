# 2. Теоретическое введение

MongoDB — это документоориентированная система управления базами данных (NoSQL). В отличие от реляционных SQL БД, где информация хранится в таблицах со строго определённой структурой столбцов, MongoDB хранит данные в виде документов. Документы объединяются в коллекции.

Документ MongoDB имеет структуру, похожую на JSON, но фактически хранится в формате BSON (Binary JSON). BSON поддерживает стандартные типы JSON, а также дополнительные типы данных: ObjectId, Date, Decimal128, Binary Data и другие.

Коллекция MongoDB является аналогом таблицы в SQL. Однако в отличие от таблицы документы одной коллекции могут иметь разные поля. Например, один документ книги может содержать отзывы пользователей, а другой — нет.

Главное преимущество документоориентированного подхода — удобное хранение сложных объектов с вложенными структурами и массивами. Например, информация о книге может содержать жанры, отзывы, характеристики и другие связанные данные внутри одного документа.

# Задание 10. Построение Aggregation Pipeline

## Постановка задачи

Получить статистику:

* только книги после 1900 года;
* группировка по жанрам;
* количество книг;
* средняя цена;
* сортировка по количеству.

---

## Этапы выполнения

```javascript
db.books.aggregate(

[

// Фильтрация документов перед обработкой
{
$match:
{
year:
{
$gt:1900
}
}
},

// Разворачиваем массив жанров
{
$unwind:"$genres"
},

// Группировка данных
{
$group:
{
_id:"$genres",

count:
{
$sum:1
},

avgPrice:
{
$avg:"$price"
}

}
},

// Сортировка групп
{
$sort:
{
count:-1
}
},

// Красивый вывод результата
{
$project:
{
Жанр:"$_id",
Количество:"$count",
СредняяЦена:"$avgPrice",
_id:0
}
}

]

)
```

---

## Требования

Использовать:

* `$match`;
* `$group`;
* `$avg`;
* `$sort`;
* `$project`.

---

## Ожидаемый результат

Получить таблицу:

```
Жанр | Количество | СредняяЦена
```

# Методические рекомендации

## 1. Правила проекции

В MongoDB нельзя одновременно использовать включение и исключение:

Нельзя:

```javascript
{
title:1,
price:0
}
```

Исключение:

```javascript
_id:0
```

разрешено.

Если нужно вывести все поля кроме одного:

Правильно:

```javascript
{
password:0
}
```

---

## 2. Разница updateOne и updateMany

`updateOne()`:

* изменяет только первый найденный документ;
* даже если условию соответствует 10 документов.

Пример:

```javascript
db.books.updateOne(
{
author:"Автор"
},
{
$set:
{
rating:5
}
}
)
```

Изменится только одна книга.

Для изменения всех используется:

```javascript
updateMany()
```

---

## 3. Работа с массивами

Операторы:

```javascript
$push
$pull
```

работают только с массивами.

Правильно:

```javascript
genres:[
"роман",
"фэнтези"
]
```

Ошибка:

```javascript
genres:"роман"
```

При использовании `$push` к строке MongoDB выдаст ошибку типа данных.

---

## 4. Порядок этапов агрегации

Aggregation Pipeline выполняется строго сверху вниз.

Неправильно:

```
$group
↓
$match
```

Правильно:

```
$match
↓
$group
```

Фильтрацию необходимо выполнять как можно раньше.

Это уменьшает количество документов для обработки и повышает производительность.
