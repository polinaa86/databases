### Задача 1. Создание коллекции и вставка документов с гибкой схемой  
**Действие:**  
1. Подключитесь к серверу MongoDB с помощью `mongosh`.  
2. Создайте (или переключитесь) базу данных `college`:  
   ```javascript
   use college
   ```  
3. Создайте коллекцию `courses` явно, если это требуется (она создастся автоматически при вставке первого документа):  
   ```javascript
   db.createCollection("courses")
   ```  
4. Вставьте пять документов, моделирующих учебные курсы колледжа. Используйте `insertMany`.  
   *Требования к документам:*  
   - Каждый документ должен содержать поля: `title` (строка), `department` (строка), `hours` (число), `description` (строка).  
   - Поле `active` (логическое) должно присутствовать **не во всех** документах — минимум у одного оно отсутствует.  
   - Поле `modules` — массив вложенных документов с полями `name` (строка) и `topics` (массив строк). Минимум у одного документа `modules` отсутствует или пуст.  
   - Дополнительное поле `prerequisites` (массив строк) добавьте только в один или два документа.  
   - Один из документов должен содержать модуль с `name: "Устаревший модуль"` (потребуется в Задаче 4).  

   *Пример структуры документов (создайте свои, соблюдая вариативность):*  
   ```javascript
   db.courses.insertMany([
     {
       title: "Базы данных",
       department: "Информационные технологии",
       hours: 72,
       description: "Курс по реляционным и нереляционным СУБД",
       active: true,
       modules: [
         { name: "Введение", topics: ["История", "Архитектура"] },
         { name: "SQL", topics: ["SELECT", "JOIN", "GROUP BY"] },
         { name: "Устаревший модуль", topics: ["Старая тема"] }
       ],
       prerequisites: ["Основы программирования"]
     },
     {
       title: "Веб-разработка",
       department: "Информационные технологии",
       hours: 60,
       description: "Создание современных веб-приложений",
       active: true,
       modules: [
         { name: "HTML/CSS", topics: ["Семантика", "Flexbox"] },
         { name: "JavaScript", topics: ["ES6", "DOM"] }
       ]
       // нет поля prerequisites и нет active (умышленно)
     },
     // ... ещё 3 документа с вариациями
   ])
   ```  
   *Важно:* обеспечьте, чтобы минимум два документа относились к одной кафедре `"Информационные технологии"` и имели `hours > 50`, это потребуется в Задаче 2.

**Ожидаемый результат:**  
В коллекции `courses` сохранены пять документов с разной структурой: некоторые без поля `active`, некоторые без `prerequisites`, у одного отсутствует массив `modules`.  

**Способ проверки:**  
```javascript
db.courses.find().pretty()
```  
Убедитесь, что документы выводятся и их структура различается. Выполните `db.courses.countDocuments()` — должно вернуть `5`.

### Задача 2. Чтение данных: фильтрация, проекция, логические операторы  
**Действие:**  
1. Найдите и выведите все курсы, которые относятся к кафедре `"Информационные технологии"` и имеют количество часов больше 50.  
   Используйте оператор `$gt`.  
   ```javascript
   db.courses.find({
     department: "Информационные технологии",
     hours: { $gt: 50 }
   })
   ```  
2. Для того же условия выведите только названия курсов и количество часов, исключив `_id`.  
   ```javascript
   db.courses.find(
     { department: "Информационные технологии", hours: { $gt: 50 } },
     { title: 1, hours: 1, _id: 0 }
   )
   ```  
3. Найдите все курсы, которые либо имеют `active: true`, либо количество часов строго равно 60. Используйте оператор `$or`.  

**Ожидаемый результат:**  
- Первый запрос возвращает только документы, удовлетворяющие обоим условиям (кафедра ИТ и часы > 50).  
- Второй запрос показывает только запрошенные поля.  
- Третий запрос выводит объединение результатов по условию ИЛИ.  

**Способ проверки:**  
Вручную просмотрите выводимые документы и убедитесь, что отобраны верно. Для третьего запроса проверьте, что в результат попали, например, курс с `hours: 60`, даже если его `active` не true.

### Задача 4. Работа с массивами и вложенными объектами: поиск и модификация элементов  
**Действие:**  
1. Найдите все курсы, в которых есть хотя бы один модуль с названием `"SQL"`, используя оператор `$elemMatch`. Выполните запрос:  
   ```javascript
   db.courses.find({ modules: { $elemMatch: { name: "SQL" } } })
   ```  
   Убедитесь, что курсы, не содержащие такого модуля, не попали в выборку.  
2. Удалите из всех документов модуль с названием `"Устаревший модуль"` (он должен присутствовать хотя бы в одном документе из Задачи 1). Используйте оператор `$pull` в `updateMany`:  
   ```javascript
   db.courses.updateMany(
     { "modules.name": "Устаревший модуль" },
     { $pull: { modules: { name: "Устаревший модуль" } } }
   )
   ```  
   Проверьте, что модуль действительно удалён: повторите поиск с `$elemMatch`, результат должен быть пустым.  
3. В курсе `"Базы данных"` переименуйте модуль `"SQL"` в `"Язык SQL"`, используя позиционный оператор `$` в `updateOne`.  
   ```javascript
   db.courses.updateOne(
     { title: "Базы данных", "modules.name": "SQL" },
     { $set: { "modules.$.name": "Язык SQL" } }
   )
   ```  
   Выведите документ и убедитесь, что название модуля изменилось.

**Ожидаемый результат:**  
- Успешный поиск курсов по вложенному полю массива.  
- Модуль `"Устаревший модуль"` отсутствует во всех документах коллекции.  
- В курсе `"Базы данных"` модуль `"SQL"` переименован.  

**Способ проверки:**  
После каждого шага выполняйте соответствующий запрос `find` и анализируйте вывод. Для удаления выполните `db.courses.find({ "modules.name": "Устаревший модуль" })` — запрос не должен возвращать документов.

### Задача 5. Создание индексов и анализ производительности  
**Действие:**  
1. До создания индексов выполните поиск всех курсов кафедры `"Информационные технологии"` с анализом плана выполнения:  
   ```javascript
   db.courses.find({ department: "Информационные технологии" }).explain("executionStats")
   ```  
   В выведенном результате найдите стадию `winningPlan`. Ожидается, что на данном этапе будет указан `COLLSCAN` (полное сканирование коллекции).  
2. Создайте индекс по полю `department`:  
   ```javascript
   db.courses.createIndex({ department: 1 })
   ```  
3. Повторите тот же запрос с `explain("executionStats")`. Убедитесь, что в `winningPlan` теперь указан `IXSCAN` по созданному индексу. Зафиксируйте изменение времени выполнения (поле `executionTimeMillis`) — оно может незначительно отличаться на малых объёмах, важна смена плана.  
4. Создайте составной индекс по полям `department` и `hours`:  
   ```javascript
   db.courses.createIndex({ department: 1, hours: 1 })
   ```  
5. Выполните запрос с сортировкой по `hours` для кафедры `"Информационные технологии"` и снова получите `explain`:  
   ```javascript
   db.courses.find({ department: "Информационные технологии" }).sort({ hours: 1 }).explain("executionStats")
   ```  
   Зафиксируйте, изменился ли план выполнения по сравнению с использованием только индекса по `department`.

**Ожидаемый результат:**  
- На этапе без индекса — `COLLSCAN`.  
- После создания индекса `department` — `IXSCAN` при фильтрации по `department`.  
- Составной индекс используется для запроса с фильтрацией и сортировкой по тем же полям.  
- Студент может описать, как меняется `executionTimeMillis` (допускается как уменьшение, так и отсутствие значительных изменений из-за малого объёма данных — объяснение этого факта приветствуется).  

**Способ проверки:**  
Предоставить фрагменты вывода `explain` с выделенными ключевыми секциями (`winningPlan` и `executionTimeMillis`). Объяснить устно или в отчёте, почему план изменился.

### Задача 6. Агрегация: группировка и вычисление статистики  
**Действие:**  
Составьте конвейер агрегации, который:  
1. Группирует курсы по полю `department`.  
2. Для каждой кафедры вычисляет:  
   - общее количество курсов (`count`: `{ $sum: 1 }`);  
   - суммарное количество часов (`totalHours`: `{ $sum: "$hours" }`);  
   - среднее количество часов на курс (`avgHours`: `{ $avg: "$hours" }`).  
3. Сортирует результат по убыванию количества курсов.  

*Пример реализации:*  
```javascript
db.courses.aggregate([
  {
    $group: {
      _id: "$department",
      count: { $sum: 1 },
      totalHours: { $sum: "$hours" },
      avgHours: { $avg: "$hours" }
    }
  },
  { $sort: { count: -1 } }
])
```  

**Ожидаемый результат:**  
Вывод агрегированных данных, где каждая строка представляет одну кафедру, количество курсов на ней, общую и среднюю нагрузку в часах.  

**Способ проверки:**  
Сравните полученные цифры с исходными данными коллекции (например, вручную посчитайте сумму часов для каждой кафедры). Убедитесь, что сортировка выполнена корректно.

## Вывод по лабораторной работе  
Лабораторная работа демонстрирует ключевые особенности MongoDB как документо-ориентированной СУБД:  
- **Гибкая схема** позволяет хранить в одной коллекции документы с разным набором полей, что упрощает эволюцию модели данных.  
- **BSON-формат** поддерживает вложенные объекты и массивы, делая возможным хранение связанных сущностей в одном документе без жёсткой нормализации.  
- Операции **CRUD** реализуются с помощью выразительного языка запросов, включающего операторы модификации массивов (`$push`, `$pull`, позиционный `$`).  
- **Индексы** кардинально меняют план выполнения запроса (с `COLLSCAN` на `IXSCAN`), что критически важно для производительности при росте объёма данных.  
- **Агрегация** предоставляет инструменты для аналитической обработки прямо в базе данных, аналогичные `GROUP BY` в SQL, но над документами.  

Полученные навыки являются основой для дальнейшего изучения MongoDB, проектирования схем и оптимизации запросов в реальных информационных системах.

## Самопроверка лабораторной работы на неточности  

1. **Соответствие структуре:** все разделы (тема, цель, оборудование, задачи, вывод, формат отчёта, вопросы, ограничения, критерии, баллы) присутствуют и раскрыты.

3. **Количество задач:** 6 задач, что соответствует требованию 5–7.  
4. **Покрытие CRUD:**  
   - Create (вставка документов) — Задача 1;

5. **Демонстрация особенностей MongoDB:**  
   - гибкая схема — Задача 1 (документы с разным набором полей);  
   - работа с массивами и вложенными объектами — Задачи 3, 4;

6. **Ограничение по времени:** 4–6 академических часов достаточно для выполнения всех шагов с учётом оформления отчёта.  
7. **Вопросы для проверки самостоятельности:** перечень из 6 вопросов покрывает теорию, использованную в заданиях.  
8. **Критерии оценки:** прозрачны, распределение баллов отражает сложность задач.
